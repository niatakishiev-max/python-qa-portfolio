# Python / QA Portfolio

Учебное портфолио начинающего Python-разработчика и QA Automation trainee.

Цель портфолио — показать практические навыки, которые можно обсуждать на собеседовании: Python, Git/GitHub, тестирование, API, автотесты и базовую проектную дисциплину.

## Ключевые навыки

- Python basics: переменные, условия, циклы, списки, словари, функции
- CLI-приложения на Python
- разделение бизнес-логики и пользовательского интерфейса
- работа с JSON-файлами
- Git и GitHub workflow
- unit-тесты на `pytest`
- `pytest.mark.parametrize`
- fixtures в `pytest`
- `monkeypatch`, `capsys`, `tmp_path`
- GitHub Actions / CI
- HTTP и REST API
- API-тестирование через `requests`
- schema validation через `jsonschema`
- управление зависимостями через `uv`, `pyproject.toml` и `uv.lock`
- ручное тестирование: чеклисты, тест-кейсы, тест-раны, баг-репорты

### Python API Testing Practice

Репозиторий: https://github.com/niatakishiev-max/python-api-testing-practice

Учебный проект для практики HTTP, REST API и API-автотестов.

Что реализовано:

- API-клиент через `requests`
- `GET`, `POST`, `PUT`, `PATCH`, `DELETE`
- проверка status code
- проверка JSON body
- проверка headers и `Content-Type`
- query params
- negative cases для неполного JSON body
- параметризация API-тестов
- schema validation через `jsonschema`
- запуск тестов через `uv run pytest`
- управление зависимостями через `pyproject.toml` и `uv.lock`
- GitHub Actions с запуском тестов через `uv`

Что показывает проект:

- понимание request / response
- базовые REST-методы
- работу с JSON API
- проверку структуры ответа
- проверку позитивных и негативных сценариев
- базовый уровень API automation
- современный подход к зависимостям через `uv`
